# Fake Review Detection Project Analysis

This document provides a comprehensive analysis and explanation of all files in the fake review detection project.

## Project Overview
This project is designed to detect fake reviews using machine learning techniques. It combines textual analysis with behavioral features from user data to classify reviews as genuine or deceptive. The project uses a hybrid model approach with Random Forest classifier and TF-IDF vectorization.

## File Structure and Analysis

### 1. __init__.py
- **Purpose**: Python package initialization file
- **Content**: Empty file (standard for Python packages)
- **Explanation**: This file makes the directory a Python package. It's empty because no package-level initialization is needed.

### 2. .gitignore
- **Purpose**: Git ignore file to exclude certain files from version control
- **Content**:
  - Python cache files (__pycache__/, *.pyc)
  - Virtual environment directories (venv/, .myenv/)
  - Large data and model files (*.pkl, data/, nltk_data/)
- **Explanation**: Prevents large or generated files from being committed to Git, keeping the repository lightweight.

### 3. data_preprocessing.py
- **Purpose**: Text preprocessing and feature extraction utilities
- **Key Functions**:
  - `preprocess_text()`: Cleans text by lowercasing, removing punctuation, stop words, and lemmatizing
  - `create_tfidf_features()`: Creates TF-IDF vector representations of text
  - `load_and_preprocess_data()`: Loads and preprocesses the deceptive opinion dataset
- **Explanation**: Handles all text cleaning and feature extraction for the traditional dataset approach.

### 4. data_preprocessing_yelp.py
- **Purpose**: Specialized preprocessing for Yelp dataset with behavioral features
- **Key Functions**:
  - `load_yelp_data()`: Loads Yelp reviews and user data, merges them, extracts features
- **Features Extracted**:
  - Textual: Review text, length
  - Behavioral: Stars, useful/funny/cool votes, user review count, average stars, elite status, friends, fans, compliments
- **Labeling Heuristic**: Uses simple rules to label potentially deceptive reviews (e.g., extreme ratings with short reviews)
- **Explanation**: Creates a comprehensive dataset combining textual and behavioral features for more robust fake review detection.

### 5. main.py
- **Purpose**: Main script that orchestrates the entire training pipeline
- **Configuration**:
  - MODEL_TO_USE: Set to 'hybrid' for the ensemble approach
  - DATASET_TO_USE: Set to 'yelp' for Yelp dataset
- **Pipeline Steps**:
  1. Load and preprocess data
  2. Extract features (TF-IDF + behavioral)
  3. Perform cross-validation
  4. Train final model
  5. Evaluate performance
  6. Sample predictions for manual review
  7. Save model and vectorizer
- **Explanation**: The central script that runs the complete machine learning pipeline from data loading to model saving.

### 6. model_training.py
- **Purpose**: Model training and evaluation utilities
- **Key Functions**:
  - `train_model()`: Trains Logistic Regression model
  - `evaluate_model()`: Prints accuracy and classification report
  - `save_model()`: Saves trained model to pickle file
- **Explanation**: Contains reusable functions for training, evaluating, and saving machine learning models.

### 7. prediction.py
- **Purpose**: Prediction utilities for new reviews
- **Key Functions**:
  - `load_assets()`: Loads saved model and vectorizer
  - `predict_fake_review()`: Makes prediction on new review text
- **Explanation**: Provides functionality to load trained models and make predictions on new, unseen reviews.

### 8. README.md
- **Purpose**: Project documentation
- **Content**: Currently empty
- **Explanation**: Should contain project description, setup instructions, and usage guide.

### 9. requirements.txt
- **Purpose**: Python dependencies list
- **Dependencies**:
  - pandas: Data manipulation
  - scikit-learn: Machine learning algorithms and tools
  - nltk: Natural language processing
  - numpy: Numerical computing
  - scipy: Scientific computing (used for sparse matrices)
  - streamlit: Web app framework
  - matplotlib, seaborn: Data visualization
  - wordcloud: Text visualization
  - pickle5: Enhanced pickling for Python objects
- **Explanation**: Lists all required Python packages for the project to run.

### 10. data/ Directory
- **Purpose**: Data storage directory
- **Contents**: 
  - deceptive-opinion.csv: Traditional deceptive review dataset
  - yelp_json/: Yelp dataset files (reviews and user data)
- **Explanation**: Contains raw datasets used for training and testing the models.

## Additional Files (Generated during execution)
- hybrid_model.pkl: Trained Random Forest model
- tfidf_vectorizer.pkl: Fitted TF-IDF vectorizer
- __pycache__/: Python bytecode cache (ignored by Git)

## Project Workflow
1. **Data Loading**: Load Yelp reviews and user data
2. **Preprocessing**: Clean text and extract behavioral features
3. **Feature Engineering**: Create TF-IDF vectors and combine with behavioral features
4. **Model Training**: Train Random Forest classifier with cross-validation
5. **Evaluation**: Assess model performance on test set
6. **Prediction**: Use trained model to classify new reviews

## Key Innovations
- **Hybrid Approach**: Combines textual analysis with user behavioral patterns
- **Yelp Dataset**: Uses real-world data instead of synthetic deceptive reviews
- **Heuristic Labeling**: Simple rules for identifying potentially fake reviews
- **Cross-Validation**: Ensures model stability and prevents overfitting

## Usage
1. Install dependencies: `pip install -r requirements.txt`
2. Run training: `python main.py`
3. Make predictions: Use prediction.py functions in your application

This project demonstrates a comprehensive approach to fake review detection using both content and context features.
